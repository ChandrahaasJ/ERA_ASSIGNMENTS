Section 1 — Activations exist for a reason

A neuron computes z = w·x + b — always a straight line. Activation functions bend that line into curves, kinks, and plateaus. Without them, even a "deep" network collapses to a single linear map.

Section 2 — Depth without nonlinearity is a lie

Same total neuron budget (16). Does it matter if they're in one wide layer or stacked deep? Yes — but only because ReLU activations between layers let each layer compose what the previous one detected.

the math — addition vs composition

Shallow (1 layer, n neurons) — values ADD:

output(x) = Σᵢ wᵢ · ReLU(wᵢ·x + bᵢ) = w₁·ReLU(…) + w₂·ReLU(…) + … + wₙ·ReLU(…)

Each neuron places one kink at a different position. The output is a sum of independent detectors. Maximum distinct linear regions ≈ n (one per neuron).

Deep (k layers, n neurons each) — values MULTIPLY in effect:

h⁽¹⁾ = ReLU(W⁽¹⁾·x + b⁽¹⁾) ← layer 1: raw input h⁽²⁾ = ReLU(W⁽²⁾·h⁽¹⁾ + b⁽²⁾) ← layer 2: combinations of layer 1 h⁽³⁾ = ReLU(W⁽³⁾·h⁽²⁾ + b⁽³⁾) ← layer 3: combinations of layer 2 output = w·h⁽ᵏ⁾ + b

Layer 2 doesn't see raw x — it sees what layer 1 already found. Each new layer can split regions that previous layers created. Maximum distinct linear regions ≈ nk (exponential in depth).

Without activations: W⁽³⁾·W⁽²⁾·W⁽¹⁾·x = Wcombined·x — still one straight (or flat) transformation. Depth is a lie without nonlinearity.
Layer 1 Detects simple patterns on raw input — edges, thresholds, kinks at specific x positions ReLU(w·x + b)
Layer 2 Combines layer-1 patterns — "edge here AND edge there" becomes a feature ReLU(Σ wᵢ·hᵢ⁽¹⁾)
Layer 3 Combines features into higher-level shapes — bumps, valleys, local curves ReLU(Σ wᵢ·hᵢ⁽²⁾)
Layer 4 Assembles global structure — full waveforms, complex boundaries Σ wᵢ·hᵢ⁽³⁾

Section 3 — How embeddings learn similarity from nothing but next-token prediction

Built from a real trained model — 17 tokens, 8-dimensional embeddings, no category labels ever shown.

1The hidden structure in the training data

The sentences were generated from a template. Click to generate a new example — notice animals can appear in either slot, but fruits only ever appear as the object.

Key structural fact: cat, dog, cow, horse, pig are all interchangeable in both slots. The model never sees a rule saying "these are animals" — it only sees that swapping them never breaks the sentence pattern.

2What "next-token prediction" actually asks

At every position, the model is only ever asked one question: given the words so far, what comes next? Pick a context below and see what the model had to predict during training.

?

3Why gradient descent pulls similar words together

Because cat, dog, pig all need to produce the same output distribution over next-tokens, the model minimizes loss most easily by giving them similar embedding vectors — so the shared output layer can treat them the same way. This is purely a side-effect of loss minimization.

The mechanism in one line: if two tokens are followed by the same distribution of next-tokens across the whole dataset, gradient descent has no reason to keep their embeddings apart — and every shared training example nudges them closer.

Section 4 — Memorization vs. generalization, and how data closes the gap

A fixed-capacity model trained on the same noisy classification task at three dataset sizes: 20, 200, 2000 examples.

3The generalization gap, directly

gap = test loss − train loss. This single number captures the whole claim: a large gap means memorization, a small gap means genuine generalization. Hover any point for the exact value.

4Why this happens

At N=20: the model has enough flexibility to wrap a tight decision boundary around each individual point — including the noisy, mislabeled ones. It reaches ~100% train accuracy, but the boundary it drew is overfit to specific points rather than the true wavy-circle pattern, so it fails badly on new points.

At N=2000: the same model capacity can no longer afford to wrap tightly around every single point — there are too many of them, and they collectively outline the true pattern. The only boundary that fits most of 2000 points reasonably well is close to the actual wavy circle. Train accuracy even drops slightly (the model can't perfectly memorize noise anymore), but test accuracy rises sharply.

The core idea: a fixed-capacity model has a limited "budget" of flexibility. With little data, that budget gets spent contouring around individual noisy points. With lots of data, that same budget can only be spent capturing the shared, repeating pattern — because that's the only thing that reduces loss across so many examples at once.
